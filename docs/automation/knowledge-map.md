# 自动化知识地图

## 1. 目标

这份文档用于给后续 AI 自动化或脚本做“路由”。

它回答的问题不是“内容怎么写”，而是：

- 不同任务应该优先读哪些文档
- 输入和输出大致长什么样
- 哪些环节可以自动化
- 哪些环节必须人工确认

## 3. 当前最适合自动化的任务

### 3.1 选题生成

输入：

- 最近已发内容
- 主题记忆库
- 爆款规律

输出：

- 新选题 3-5 个
- 每个选题对应的内容形式建议
- 是否重复、重复在哪里、递进点是什么

人工确认点：

- 最终选题是否符合当下账号节奏

### 3.2 文案初稿生成

输入：

- 已确认的主题
- 目标形式
- 历史相关篇目
- 生产规范

输出：

- `content/DayN.md` 初稿
- 置顶评论建议
- 标签建议

人工确认点：

- 人设是否稳定
- 是否重复旧表达
- 是否需要更换结构

### 3.3 评论回复建议

输入：

- 评论正文
- `comment-library.md`

输出：

- 评论分类
- 风险等级
- 回复建议 1-2 条

人工确认点：

- `R1` 及以上内容
- 含争议、冲突、现实危机的评论

### 3.4 配图页稿生成

输入：

- 已定稿文案
- 配图规范
- 旧篇设计参考

输出：

- 每页文案拆分
- Canva 落版建议

人工确认点：

- 页数是否过长
- 视觉节奏是否自然

## 4. 推荐字段约定

为了方便后续做稳定的自动化，建议几类任务都尽量使用固定字段。

### 4.1 选题任务

| 字段 | 含义 |
|------|------|
| `topic_id` | 主题 ID |
| `topic_title` | 选题标题 |
| `content_type` | 内容形式 |
| `related_days` | 相关历史篇目 |
| `progression_point` | 递进点 |
| `risk_note` | 人设或重复风险 |

### 4.2 评论任务

| 字段 | 含义 |
|------|------|
| `category_id` | 评论类别 |
| `risk_level` | 风险等级 |
| `reply_mode` | 回复方式 |
| `emotion` | 主情绪 |
| `reply_goal` | 回复目标 |
| `draft_reply` | 回复草稿 |

### 4.3 配图任务

| 字段 | 含义 |
|------|------|
| `page_index` | 页码 |
| `page_text` | 该页文字 |
| `layout_note` | 版式说明 |
| `reference_design` | 参考旧稿 |

## 5. 自动化边界

下面这些事适合自动化：

- 检索历史主题
- 判断重复风险
- 给出文案初稿
- 生成评论回复建议
- 生成配图页稿
- 输出复盘草稿

下面这些事不建议全自动：

- 直接发送评论回复
- 自动发布内容
- 处理粉圈争议评论
- 处理高风险情绪评论
- 最终拍板账号阶段策略

---

## 6. 最小可行自动化顺序

如果后续要开始真的接自动化，建议按这个顺序推进：

1. 先做评论分类 + 回复建议
2. 再做选题判重 + 递进建议
3. 再做文案初稿生成
4. 最后再接配图页稿和数据复盘

原因：

- 评论任务规则最清楚，最容易标准化
- 选题任务依赖已有知识库，投入产出比高
- 文案生成需要前两者先稳定
- 配图和复盘更适合在前面流程跑顺后再接
