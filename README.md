## 二、内容体系

### 2.1 三大内容板块

| 板块 | 作用 | 示例主题 |
|------|------|----------|

### 2.2 内容形式多样化（避免模板同质化）

以往采用固定结构（提问-描述-概念-解释-结尾），虽稳定但易显重复。现鼓励根据主题灵活选择形式：

| 形式 | 适合场景 | 示例 |
|------|----------|------|
| 故事型 | 有具体经历可分享 | 《为了见他，我吃了三个月泡面》 |
| 对话型 | 呈现内心拉扯或粉丝问答 | 《闺蜜问我：你追星花了多少钱》 |
| 清单型 | 提供干货建议 | 《追星省钱的5个方法》 |
| 对比型 | 制造反思张力 | 《给他花的钱，够给自己报个课了》 |
| 情绪型 | 单纯感性抒发 | 短文本，不强行给方法 |
| 问答型 | 回应粉丝共性提问 | 以回复粉丝切入展开 |

每篇内容仍需保持“温柔理性”底色，但结构可随内容流动。

### 3.3 问题诊断

- **缺少成长线**：20篇中“我”的状态几乎平铺，没有起伏和递进，真实感弱。
- **选题重复**：同一主题（如“喜欢可以很轻”）反复出现但未深入。
- **人设轻微漂移**：部分篇目“我”的定位不一致（已想通的前辈 vs 正在上头的人）。
- **节日选题表现差**：过年系列数据低，需控制时效性内容比例。
- **模板同质化风险**：长期使用固定结构易让读者产生审美疲劳，需引入更多元的内容形式。

## 六、粉丝互动原则与模板（精简版）

### 6.1 互动原则

1. **共情优先**：开头先表达理解（“懂你”“我也有过”）
2. **不评判不站队**：不批评其他粉丝或粉圈
3. **引导自我思考**：简短提问（“后来怎么走出来的？”“现在感觉怎么样？”）
4. **温柔收尾**：加一个表情符号（💛✨）
5. **回复简洁**：避免长篇大论，点到即止，保留余韵

### 6.2 常用回复模板库（精简示例）

| 场景 | 回复模板 |
|------|----------|
| 表达共鸣 | “懂你，我也有过。后来发现……”（1-2句） |
| 询问方法 | “我之前试过XX，你也可以试试～” |
| 倾诉负面情绪 | “抱抱，这种情绪很正常。我在。” |
| 分享美好瞬间 | “这个瞬间好温柔，谢谢你分享。” |
| 追问细节 | “当时是什么让你走出来的？” |

## 八、自动化流程设计思路

### 8.1 可自动化的环节

| 环节 | 当前方式 | 自动化目标 |
|------|----------|------------|
| 选题 | AI规划 | 建立选题规则库+记忆库，AI可基于规则生成选题 |
| 内容撰写 | AI写+人工审核 | 提供形式库+历史内容记忆，AI生成初稿 |
| 粉丝互动 | AI辅助回复 | 建立精简回复模板库+情绪识别规则，AI生成回复建议 |
| 数据分析 | 人工统计 | 定期导入数据，AI产出复盘报告 |
| 产品预热 | 人工设计 | AI根据产品框架生成预热文案 |

### 8.2 需要沉淀的知识库

- 内容记忆库（主题-篇目-深度）
- 爆款要素库（什么容易火）
- 内容形式库（故事、对话、清单等模板）
- 精简互动回复模板库
- 人设话术库（常用表达、金句风格）
- 选题递进规则（如何避免重复）
